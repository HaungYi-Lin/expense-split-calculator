# 多人花費分攤與轉帳結算工具

這是一個生活化的多人花費分攤網頁工具，可用於聚餐、出遊、合購、住宿、交通費、宿舍生活與朋友代墊費用等情境。作品使用純前端 HTML、CSS、JavaScript 製作，不需要後端與資料庫，可直接部署到 GitHub Pages。

本作品採用同一個 `index.html` 內的多層頁面切換設計。首頁提供三個功能入口，點選後會在同一個網頁內切換到對應功能頁面，不會另開視窗，也不會跳到其他 HTML 檔案。

## 功能特色

- 簡易平均分攤
- 個人消費分攤
- 多人付款結算
- 服務費計算
- 折扣分攤
- 多人代墊處理
- 最少轉帳次數建議
- 可複製 LINE 結算文字
- 支援手機與電腦瀏覽
- 支援 GitHub Pages

## 使用方式

1. 開啟首頁 `index.html`
2. 選擇需要的分攤模式
3. 在同一網頁內進入對應功能頁
4. 輸入金額資料
5. 按下計算或結算按鈕
6. 查看摘要、明細與轉帳建議
7. 複製結算文字並貼到 LINE 群組

## 三種模式說明

### 簡易平均分攤

適合一個人先付款，其他人平均分攤的情境，例如一般聚餐、飲料、合購。可輸入總金額、人數、服務費、折扣與付款者名稱，系統會計算每人應付金額與其他人應轉給付款者的金額。

### 個人消費分攤

適合每個人消費金額不同的情況，例如有人吃比較貴、有人只喝飲料。服務費與折扣會依照每個人的個人消費比例分攤，並依照已付款金額計算誰應收、誰應付。

### 多人付款結算

適合多人都有代墊費用的情況，例如出遊住宿、交通費、門票、餐費由不同人先付款。使用者可直接輸入每個人的應付金額與已付款金額，系統會計算差額並產生最少轉帳次數建議。

## 計算邏輯

服務費金額會依照總金額或個人消費比例計算。折扣金額在個人消費分攤模式中會依照消費比例分攤。

差額計算方式為：

```text
差額 = 已付款金額 - 實際應付金額
```

差額大於 0 代表此人應收錢，差額小於 0 代表此人應付錢，差額等於 0 代表已結清。

最少轉帳次數會先找出所有應收者與應付者，再讓應付者依序付款給應收者，每次轉帳金額取雙方剩餘金額的較小值，直到所有差額結清。

## GitHub Pages 部署方式

1. 到 GitHub 建立新的 Repository
2. 將 `index.html`、`style.css`、`script.js`、`README.md` 上傳到 Repository
3. 進入 Repository 的 `Settings`
4. 點選 `Pages`
5. Source 選擇 `Deploy from a branch`
6. Branch 選擇 `main`，資料夾選擇 `/root`
7. 儲存後等待 GitHub Pages 產生網站連結
8. 開啟網站連結後，首頁會顯示三個功能入口
